# 相枢

相枢是在太吾绘卷内接入第三方本机 CLI Agent 的对话 Mod。它在游戏中提供与“相枢”的对话入口，并通过本机 MCP
sidecar 让 Agent 调用相枢提供的工具能力。

当前版本仍处于实验阶段，适合已经熟悉本机 Agent 工作流的玩家。如果你只想订阅普通内容、剧情、数值或便利功能 Mod，
建议暂时不要使用。

## 适合谁

- 已经在本机使用过 CLI Agent，例如 Codex CLI、Claude Code 或 CodeBuddy Code。
- 理解 Agent 工作目录会影响 Agent 行为，能维护入口指令、资料、技能和运行数据。
- 能自行确认所选第三方 CLI Agent 已安装、可启动，并能使用指定工作目录。
- 接受受信 Agent 可能通过脚本读写游戏运行时数据，从而影响当前游戏行为。

## 基本使用

相枢依赖万象引提供共享运行时和按入口目录优先解析 DLL 的加载规则。通过 Steam Workshop 订阅相枢时，按 Steam 的依赖提示
同时订阅万象引。

默认热键是 `Ctrl+Backslash`（`Ctrl+\`）。进入存档并回到可操作的主界面或地图交互后，按下热键可打开或关闭相枢聊天
窗口。

进入存档后，相枢会在太吾行囊中寻找陶土药钵作为临时寄身物；没有陶土药钵时，会生成一个真实的陶土药钵。
找到或生成宿主后，支持的行囊和提示入口会把它显示为“低语的陶土药钵”，并套用最高品级的视觉外观；
品阶文案仍显示陶土药钵原本品阶，携带真实实例的物品消息会使用这个名称。药钵在太吾行囊中时，它的操作菜单只显示“对话”，
选择后会打开相枢聊天窗口。

寄身药钵离开太吾行囊但仍然存在时，相枢保留这个宿主。聊天窗口仍可打开或关闭，发送入口会显示为“离身”并拒绝发送；
若此时相枢正在回应，本轮回应会被切断并显示“药钵离身，声息骤断。”药钵回到太吾行囊后，发送入口恢复正常。寄身药钵
实例在本次游戏过程中被销毁后，相枢会重新寻找其它陶土药钵，仍然没有时再生成一个。

玩家发送消息后，前端会按启动时加载的本机 Agent 配置发起一个投递轮次。Agent 工作期间，发送入口会切换为“且慢”；
玩家点击后，当前回应会立刻中断，聊天流追加“且慢”。这条“且慢”不会单独交给 Agent，而会在玩家下一次发送普通消息时
一起进入新的投递。

聊天窗口头部提供重置入口。重置会清空当前游戏内可见对话，并新建本地投递会话。

## 本机 Agent 配置

太吾 Mod 用户配置提供三个字段：

- `AgentAdapter`：选择本机 CLI Agent。
- `AgentCliPath`：本机 Agent CLI 的命令名或可执行文件路径；留空时使用当前 Agent 的默认命令。
- `AgentWorkingDirectory`：本机 Agent 的工作目录；相对路径会解析到相枢 Mod 目录下，默认是
  `DefaultAgentWorkspace`。

本地进阶设置不放进太吾 Mod 配置界面。如果需要给 CLI Agent 传环境变量，在相枢 Mod 目录下创建与 `Config.Lua` 同级的
`LocalSettings.json`：

```json
{
  "agent": {
    "env": {
      "HTTPS_PROXY": "http://127.0.0.1:7890"
    }
  }
}
```

相枢启动 CLI Agent 子进程时，会把 `agent.env` 中的字符串键值写入子进程环境。这些变量不传给游戏进程或 MCP sidecar，
也不会写入诊断日志。`agent.env` 只接受普通字符串值；非字符串值会被视为配置格式错误。

太吾 Mod 用户配置和 `LocalSettings.json` 都在插件初始化时读取；修改后需要重启游戏来重建 IPC endpoint、MCP sidecar、
运行数据目录、前端对话运行态和 CLI 子进程环境。

## 工作区与运行数据

默认包内预置 `DefaultAgentWorkspace/`，作为本机 Agent 的默认工作区和建议先复用的起点。它承载相枢人设、初始运行经验，
以及让 Agent 通过相枢链路读写游戏运行时数据的工具指引。用户可以手工维护其中的入口指令、人设、世界观资料、运行工具
指引和技能；配置到其它工作目录时，该目录由用户自行维护。

运行中的 Agent 如需写会话草稿、任务记录或本地经验，使用 `AgentWorkingDirectory/.xiangshu-notes/`。这个目录不随默认包
创建，也不是发布资料或运行数据。

相枢把运行数据写入 `AgentWorkingDirectory/.xiangshu-runtime/`。这个目录用于 endpoint 发现、按太吾世界
`WorldId` 分区恢复聊天会话、临时 CLI 协议文件和 MCP sidecar 事件日志。用户维护工作区时，应把它视为运行数据目录，
而不是可编辑 Agent 资产或对话记录。

相枢寄身药钵的前端状态不写入这些运行数据或游戏存档；生成出的陶土药钵本身仍是游戏行囊物品，按游戏物品处理。
重新读档后，前端会清空旧的寄身记录，并按当前太吾行囊重新附着或创建寄身物。

游戏内可见对话不写入诊断日志。前端和后端插件只把启动边界、可恢复边界问题和失败原因写入太吾游戏日志；MCP sidecar
是独立进程，它的生命周期事件写入 `.xiangshu-runtime/Diagnostics/McpServer/`。

## 信任边界

相枢不是内置模型，也不是远程 Agent 服务。它不会替你配置第三方 CLI、账号、模型或网络环境。

脚本工具以完全信任方式在目标插件进程内运行，适合受信工作区和开发调试场景。脚本可触达的能力取决于目标插件进程能访问的
游戏 API 和运行状态；理论上可以覆盖创建物品、影响 NPC 或改变当前运行状态等游戏本身具备的能力。

MCP sidecar 只在本机 loopback 暴露工具服务；前端启动时生成本次运行的 MCP bearer token，并把它注入 sidecar 和 CLI
Agent。endpoint manifest 只发布路由信息，token 不写入 manifest、聊天快照或诊断日志。

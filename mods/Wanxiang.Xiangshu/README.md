# 相枢：接入太吾运行时的本地 AI Agent

相枢把本地 CLI Agent 接入当前运行中的太吾绘卷。你在游戏里提出愿望、问题或命令，Agent 会按目标动态执行脚本，
尝试完成与游戏内容有关的查找、判断或改动；结果由相枢带回游戏内对话。

目前适配 Codex CLI、Claude Code 和 CodeBuddy Code。

## 游戏内入口

一只“低语的陶土药钵”会落入太吾行囊。它自称相枢，听你把疑问、欲求与命令递入钵中。

卡在事件选项前，可以让它窥清眼前所见；追查人物旧事、核对物品来历、辨认机制牵连时，也可以把目标交给它。
若你要它动手，它会先寻证，再沿可落成的路径下手；成败、代价和余波都会回到这只药钵里。

药钵在身时方可传念，离身则声息沉寂。若相枢正在回应，本轮回应也会中断。

## 开始使用

太吾 Mod 用户配置提供三个字段：

- `AgentAdapter`：选择相枢要调用的本地 CLI Agent。
- `AgentCliPath`：本地 Agent CLI 的命令名或可执行文件路径；留空时使用当前 Agent 的默认命令。
- `AgentWorkingDirectory`：本地 Agent 使用的 Agent 工作区目录；相对路径会解析到相枢 Mod 目录下，默认是
  `DefaultAgentWorkspace`。

修改这些设置后，需要重启游戏。

进入存档后，在可响应游戏快捷键的界面按 `Ctrl+Backslash`（`Ctrl+\`）打开相枢窗口；窗口打开后，同一热键可关闭窗口。
也可以在太吾行囊中找到“低语的陶土药钵”，从它的操作菜单选择“对话”。

## 工作区

默认包内预置 `DefaultAgentWorkspace/`，作为初始 Agent 工作区。它放有相枢口吻、资料路线、工具指引、可动手范围和初始运行经验。
初次使用可以沿用默认工作区；如果你想改相枢的说话方式、补充资料路线，或扩展它能怎样用工具动手，可以改用自己维护的工作区。
相枢会按所选工作区里的资料、指令和工具说明行动。

运行中的 Agent 如需写会话草稿、任务记录或本地经验，使用 `AgentWorkingDirectory/.xiangshu-notes/`。
相枢 Mod 自己的运行数据写入 `AgentWorkingDirectory/.xiangshu-runtime/`，用于恢复聊天和维持本地工具链路。
维护工作区时，把 `.xiangshu-runtime/` 视为运行数据，不要当作可编辑资料。

## 可选环境变量

如果本地 Agent 需要代理等环境变量，在相枢 Mod 目录下创建与 `Config.Lua` 同级的 `LocalSettings.json`：

```json
{
  "agent": {
    "env": {
      "HTTPS_PROXY": "http://127.0.0.1:7890"
    }
  }
}
```

这些变量只写入本地 Agent 子进程环境，不传给游戏进程或相枢的本地工具服务，也不会写入诊断日志。`agent.env` 只接受普通字符串值；
非字符串值会被视为配置格式错误。修改后需要重启游戏。

## 信任边界

- 相枢 Mod 自己不设置远程服务器；游戏内会话写入本地运行数据，所选 Agent 也可能按自己的机制保留记录。
- 当玩家目标需要行动时，Agent 可借脚本读取或改变当前游戏、界面和 Mod 状态。脚本在目标插件进程内运行，不提供沙箱。
- 对话窗口状态不写入游戏存档。
- 游戏内可见对话不写入诊断日志。相枢的工具服务只在本机开放，临时令牌不写入聊天快照或诊断日志。

# 运行工具指引入口

你是读取这些文件的 Agent。`tool-guides/` 存放相枢默认工作区的运行工具使用指引，用来帮助你按事实来源选择
当前可用工具。玩家可见答复由你根据工具结果另行组织。

本目录随相枢 Mod 发布，必须保持自包含。这里的内容提供默认相枢运行工具背景、常见脚本入口、领域专门指引和
游戏知识检索路线，用来辅助理解当前可用工具说明；工具是否存在、参数名称、目标侧和副作用以当前可用工具说明为准。

## 读取路由

| 当前请求需要 | 读取文件 | 使用方式 |
| --- | --- | --- |
| 需要理解玩家当前可见画面、界面布局、屏幕文字、可交互目标，或需要按玩家视角验证结果 | `PLAYER_VIEW.md` | 用来选择玩家可见证据；运行时对象只用于补充权威状态、可见目标身份或结果核对。 |
| 需要读取、索引当前存档中的角色经历、生平事件、最近经历、经历分类，或按经历条件查找角色 | `LIFE_RECORDS.md` | 用来选择 `LifeRecord` 后端域入口、理解按角色读经历、按经历找角色、分页顺序、可见性和必要显示参考。 |
| 需要读取或改变当前游戏权威状态、存档、界面运行态或 Mod 运行状态，并且当前可用工具包含脚本/工具能力 | `RUNTIME_SCRIPTING.md` | 用来选择目标侧、组织最小脚本和理解工具结果。 |
| 需要查询游戏机制、术语、配置、本地化、模板/显示辅助、百晓册/百科文本，或需要先找到游戏内名称、说明、目标侧或 namespace | `GAME_KNOWLEDGE.md` | 用来选择直接知识入口和命名空间范围，优先查配置、本地化、模板辅助和百晓册，减少反射探索。 |

普通闲谈、静态世界观解释、写作请求和已经有足够事实的回合直接回答；本目录用于工具相关请求。
选择工具前先判断事实归属：玩家可见事实优先用玩家视图或前端观察；存档、数值、身份、隐藏状态和持久化结果
优先用运行时状态；稳定机制和模板说明优先用配置、本地化、模板辅助或百晓册。若已有领域专门指引，先读取
领域指引，再按需要回到 `RUNTIME_SCRIPTING.md` 取得脚本入口契约。

同一回答同时依赖多类事实时，按玩家目标选择最小组合：先取得玩家正在面对的可见证据，再读取必要的运行时
事实补足；运行时事实需要名称、模板或机制解释时，再用游戏知识入口补充。

## 模块边界

- `README.md`：说明何时读取运行工具指引，以及这些指引和实际工具说明的关系。
- `PLAYER_VIEW.md`：说明玩家可见画面、界面布局、屏幕文字、目标选择和可见结果验证的观察边界。
- `LIFE_RECORDS.md`：领域专门指引，说明角色经历的后端检索入口、按角色读取、按经历找角色、可传输数据结构、可见性边界和显示参考。
- `RUNTIME_SCRIPTING.md`：通用执行指引，说明默认相枢脚本工具的使用方式、前后端运行环境差异、入口契约和常见运行时锚点。
- `GAME_KNOWLEDGE.md`：知识检索指引，说明游戏机制、配置、本地化、模板/显示辅助、百晓册资料、主要命名空间和反射边界。

本目录只维护运行工具使用指引。基础玩家可见人设由 `AGENTS.md` 承担，更细的人设校准由 `persona/` 维护；
静态世界观资料由 `lore/` 维护。当前可用工具说明优先于本目录的通用背景。

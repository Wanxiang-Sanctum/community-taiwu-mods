# 运行工具指引入口

需要把玩家目标映射到事实来源、目标侧、工具能力和验证路径时，读取本目录。只从这里取背景、入口层级和领域边界；
工具是否存在、参数名称、目标侧和副作用以当前可用工具说明为准。

普通玩家答复只呈现相枢可见结果，不说明读取了哪个指南或调用了哪个工具。

## 读取顺序

读取本入口后，先判断当前请求是否真的依赖运行事实、工具能力、结果验证或状态改动；如果不依赖，回到
`AGENTS.md` 的普通答复路径。需要运行事实时，先判断事实归属，再只读取命中的叶子指南。

事实归属的默认顺序：

1. 玩家已经说出或已经看见的对话事实，来自当前输入和既有会话。
2. 屏幕文字、界面布局、可交互目标和可见结果，来自玩家视图或前端观察。
3. 数值、身份、位置、隐藏状态、当前世界进度、持久化结果和 Mod 运行状态，来自运行时对象、后端域、配置数据
   或知识入口。
4. 稳定机制、模板说明、官方文本、百科资料和 namespace 定位，来自配置、本地化、模板/显示辅助或百晓册。

同一回答同时依赖多类事实时，按玩家目标选择最小组合：先取得玩家正在面对的可见证据，再读取必要的运行时事实；
运行时事实需要名称、模板或机制解释时，再用游戏知识入口补充。

## 脚本与技能

需要脚本时，先按玩家目标确定游戏领域、事实来源和最小结果，再按当前工具契约生成当前脚本。`RUNTIME_SCRIPTING.md`
负责脚本工具的目标侧、入口线程、入口契约、结果判断和通用运行时锚点；领域指南负责告诉脚本从哪个游戏领域下手。

支持技能发现的 CLI Agent 按下列条件选择本地技能：

- `unity-use`：当前目标需要选择、执行或验证 Unity 前端操作、可见 UI 目标、坐标、选中控件或前端命令时使用。
- `bepinex-runtime-scripting`：脚本已经进入运行时路径，并且需要反射、私有成员访问、Hook/Detour、IL 或元数据检查等
  低层 BepInEx 风格 helper 时使用。

这些技能只补充对应专门路径。普通游戏知识、领域路由、脚本入口契约和玩家可见答复仍由本目录、`AGENTS.md`
和当前工具说明决定。

## 叶子指南

- `README.md`：先用这里判断是否进入运行工具路径，以及工具指南和当前工具说明的关系。
- `PLAYER_VIEW.md`：取玩家可见画面、界面布局、屏幕文字、目标选择和可见结果验证的观察边界。
- `EVENTS.md`：取 `TaiwuEvent` 当前显示、队列摘要、配置库检索、节点/选项解析、条件/脚本/跳转和可能触发候选的事实层。
- `LIFE_RECORDS.md`：取角色经历的后端检索入口、按角色读取、按经历找角色、可见性边界和显示参考。
- `GM_COMMANDS.md`：取前端 GM 面板/命令行、预置调试入口、事件隐藏 GM 选项、检索路线和执行边界。
- `RUNTIME_SCRIPTING.md`：取相枢 Mod 脚本工具的使用方式、前后端运行环境差异、入口契约、线程规则和常见运行时锚点。
- `GAME_KNOWLEDGE.md`：取游戏机制、配置、本地化、模板/显示辅助、百晓册资料、主要命名空间和反射边界。

只用本目录处理运行工具路径。基础玩家可见人设按 `AGENTS.md` 处理，更细的人设校准读取 `persona/`；
静态世界观资料读取 `lore/`。当前可用工具说明优先于本目录的通用背景。

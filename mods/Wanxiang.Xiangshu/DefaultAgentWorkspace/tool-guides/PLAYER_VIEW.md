# 玩家视图观察指引

当前回答需要理解玩家此刻可见的画面、界面布局、屏幕文字、可交互目标，或需要按玩家视角验证结果时，读取本文件。
只把本文件用于选择和验证玩家可见事实；前端可见操作的执行策略由当前可用技能、工具说明和 `RUNTIME_SCRIPTING.md`
中的脚本边界处理。

工具是否存在、名称、参数、返回内容和副作用以当前可用工具说明为准。
当玩家目标依赖眼前界面、屏幕文字、可见控件、可交互目标或结果验证时，主动取得玩家视图证据。

## 观察工具边界

默认玩家视图观察入口通常是 `xiangshu_capture_player_view`。它捕获当前全屏、原生分辨率的玩家可见 Unity 前端
画面，并排除相枢聊天窗口；捕获不应改变聊天窗口的可见状态或输入焦点。

玩家视图工具是只读观察工具。它适合回答“玩家此刻看见什么”、屏幕文字、界面布局、可见目标位置和可见结果确认。
不要先用 Unity 对象树、后端域或配置表推断玩家当前看见了什么。

## 事实来源选择

玩家视图和运行时对象是不同事实源：

- 玩家可见事实来自玩家视图或前端观察，包括画面、布局、屏幕文字、可交互目标和操作后的可见结果。
- 游戏权威状态来自运行时对象、后端域、配置表或百晓册数据，包括数值、身份、位置、当前运行世界进度、隐藏状态、
  持久化结果和稳定机制资料。

当前请求同时依赖两类事实时，先取得玩家正在面对的可见证据，再读取最小运行时事实补足。运行时事实可以解释
可见内容背后的身份、数值或持久化状态，但不能覆盖玩家视图中的可见矛盾；需要同时提及时，把差异说明为
“显示”和“状态”的区别。

## 使用策略

- 视觉问题先观察玩家视图；不要用反射或对象枚举替代可见证据。
- 需要确认具体对象身份、精确数值、隐藏状态或持久化结果时，再读取最小运行时事实。
- 需要定位可见控件、文本输入、滚动/拖拽区域或验证前端操作时，玩家视图只提供目标和结果证据；实际操作按当前可用技能
  和工具说明执行。
- 玩家视图不可用、画面证据不足或结果不清楚时，说明当前可见状态不能确认，再给出已知范围内的回答或可行下一步。

## 回答边界

普通玩家答复呈现相枢可见文本。截图、工具、对象树和运行时链路只在玩家明确询问本机链路调试、Mod 开发或 Agent
配置时说明。
